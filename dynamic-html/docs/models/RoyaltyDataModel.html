RoyaltyDataModel

  • songId : String
    Song ID
  • songTitle : String
    If the sale occurred as a song purchase and not an entire album, the song name that was purchased shows (see note above for Release Title)
  • label : String
    The label listed on the release, song or ringtone
  • postedDate : Date
    The date that the report appeared in your account YYYY-MM-DD formate
  • storeName : String
    The digital store that paid you for the reported sales period
  • countryofSale : String
    The territory of the store where the sale occurred
  • perfSource : String
    The source of performance like RADIO | APPLE FAMILY | FACEBOOK | GOOGLE PLAY RAD etc
  • unitsSold : Integer
    The no of of albums, songs or ringtones sold in a reporting day
  • perUnitPrice : Double
    The price of 1 unit that sold in that stated country's currency
  • totalEarned : Double
    The amount you received after the stated exchange rate
  • currency : String
    This will always be USD since we pay out in US Dollars only
  • releaseType : String
    album, song or ringtone (a single is reported as a song sale)
  • releaseTitle : String
    The title of the album, song or ringtone
  • upc : String
    Universal Product Code
  • optionalUPC : String
    Optional Universal Product Code
  • optionalISRC : String
    Optional International Standard Recording Code(12-character, alphanumeric code)
  • salesType : String
    Download or Stream
  • extensions : Map
    Extensions column for additional fields
